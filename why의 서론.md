### 왜 의성군이 이러한 프로젝트를 제안요청했는가.

## 의성군 RFP와 ‘AI G3 도약’ 정책의 직접 연결 분석

제안요청서(RFP) **Ⅰ. 사업개요**의 “추진 배경 및 필요성”에 명시된 내용이 핵심입니다:

> “(대외) 정부 AI 정책기조에 따른 공공부문 행정혁신 가속화  
> - 정부 ‘**AI G3 도약**’을 목표로 공공부문 전반의 인공지능 도입으로 행정 업무방식과 대국민 서비스의 혁신을 핵심 과제로 제시  
> - 지방정부 역시 생성형 AI를 활용한 행정지원 체계 마련이 요구됨”

이는 2026년 현재 **정부의 공식 국정과제**를 그대로 인용한 것입니다.  
2026년 정부는 “AI G3 도약 원년”을 선언하고, AI 예산을 전년 대비 3배(약 9.9조 원)로 확대했습니다. 대통령 직속 **국가인공지능위원회**를 컨트롤타워로 삼아 2026~2028년 3개년 계획을 확정했습니다. 주요 골자는 **범정부 AX(Admin eXperience)**, 즉 공공부문 전반의 AI 전환입니다.

---

## 의도와 원하는 사항 (RFP가 추구하는 그림)

RFP는 단순히 “AI 챗봇 하나 만들자”가 아닙니다.  
**국가 AI G3 도약 전략의 지방정부 실현판**으로 설계되었습니다. 핵심 의도는 다음과 같습니다:

### 1. 소버린(주권) AI + 하이브리드 구조 확보

- **직원용(내부)**: 온프레미스 45B 이상 파라미터, 128K 컨텍스트 한국어 모델 필수 + 내부 데이터 RAG + Orchestrator/Policy Router  
  → 내부 행정자료(군정백서, 조례, 회의록 등)는 **절대 외부 전송 금지** (FUR-04)

- **대민용(외부)**: DMZ/G-클라우드 기반, 공개정보 중심 RAG + Knowledge Graph + Agent

- **비증거 업무**에만 제한적으로 외부 LLM 연결 허용  
  (관리자 승인 + 비식별화 + 단방향 망연계)

→ 이는 **국가 소버린 AI 전략**과 정확히 일치합니다.

---

### 2. LLMOps + Agentic AI로 행정 혁신 실현

- LLMOps 플랫폼으로 모델 생명주기·Fine-tuning·RAG·가드레일·로그·비식별화 전부 통합 관리 (FUR-02)

- Agent 기반 실행 구조로 실제 업무 자동화  
  (시설예약, 요금조회, 인수인계 문서 생성 등 최소 5건 이상)

- Catastrophic Forgetting 방지 Continual Learning  
- Hybrid RAG(벡터+Graph) 요구

→ 핵심: **AI 일상화 (AX)**

---

### 3. 국가 공통기반과의 미래 연계 준비

- 현재: 완전 분리 아키텍처
- 미래: 공통기반 연계 가능

→ 데이터 표준 / 인터페이스 / LLMOps / Agent API 구조로 설계됨
