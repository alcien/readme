# 대민용 AI 비서 개발 요구사항 정의서 (RD)

**문서 버전:** v0.1  
**기준 문서:** 국내 공공기관 대민용 AI 비서 운영 현황 분석 보고서

---

## 1. 개요 및 목적

### 1.1 배경
국내 공공기관의 대민용 AI 비서 도입 현황을 분석한 참조 문서를 기반으로, 신규 대민용 AI 비서 시스템 개발 시 반드시 구현해야 할 기능 및 UI/UX 요구사항을 체계적으로 정의한다.

### 1.2 참조 시스템

| 참조 시스템 | 운영 주체 | 참조 목적 |
| :--- | :--- | :--- |
| **국민비서 구삐** | 행정안전부 | 알림·푸시 기능, 범부처 연동 구조 |
| **혜택알리미** | DPG위원회 | 선제적 복지 매칭, 원스톱 처리 플로우 |
| **정부24+** | 행정안전부 | 자연어 검색, 대화형 민원 발급 UX |
| **서울톡** | 서울특별시 | 메신저 기반 챗봇 UI, 민원 접수 플로우 |
| **코레일 AI챗봇** | 한국철도공사 | 음성 AI, 배리어프리 인터페이스 |
| **인천 AI여행비서** | 인천광역시 | 멀티모달·초개인화 큐레이션 |

## 2. 시스템 기능 요구사항 (Functional Requirements)

### 2.1 핵심 기능 영역 분류
* **FR-01** 알림·푸시 서비스
* **FR-02** 맞춤형 혜택 매칭 및 선제 안내
* **FR-03** 자연어 기반 민원 검색 및 안내
* **FR-04** 대화형 민원 처리 (Conversational UX)
* **FR-05** 음성 인터페이스
* **FR-06** 원스톱 신청·처리·결과 확인
* **FR-07** 마이데이터 연동 및 정보 자기결정
* **FR-08** 외부 플랫폼 API 연계
* **FR-09** 배리어프리 (접근성)
* **FR-10** 보안 및 데이터 거버넌스

| :--- | :--- | :--- | :---: |

| :--- | :--- | :--- | :---: |

| **FR-04-02** | 대화형 민원서류 발급 | 텍스트 타이핑 없이 AI와의 대화(텍스트·음성)만으로 민원 서류 발급 프로세스 완결 | **필수** |

## 3. UI/UX 요구사항 정의서

> **참조 벤치마크:** 서울톡 (카카오톡 채널 기반 챗봇), 국민비서 구삐 (다채널 알림 UI)

### 3.1 전체 UI 철학
| 원칙 | 정의 |
| :--- | :--- |
| **생활 밀착형 (Parasitic UX)** | 별도 앱 설치 부담 최소화. 사용자가 이미 쓰는 메신저·금융 앱 내에서 서비스 소비 가능 |
| **대화 중심 (Conversational First)** | 메뉴 탐색이 아닌 대화창 입력이 모든 서비스의 시작점 |
| **선제적 노출 (Push-first)** | 사용자가 찾기 전에 시스템이 먼저 필요한 정보를 화면에 노출 |
| **단계 최소화 (Zero Friction)** | 목표 달성까지의 화면 전환·입력 필드·인증 절차를 최대한 압축 |
| **포용적 설계 (Inclusive by Default)** | 고령층·장애인·외국인을 처음부터 고려한 디자인. 별도 '접근성 모드'가 아닌 기본값 |
