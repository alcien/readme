# 의성군 AI 플랫폼 사업 분석 정리
> (AI G3 도약 기반 RFP 의도 및 아키텍처 해석)

---

## 1. 사업 배경: AI G3 도약과의 직접 연결

### 1.1 정부 정책 기반
의성군 RFP의 추진 배경은 단순 사업이 아니라 국가 AI 전략의 하위 실행입니다.

**핵심 정책 방향**
* AI G3 (Global Top 3) 달성
* 공공부문 AI 전면 도입 (AX: Admin eXperience)
* 국산 LLM 및 주권형 AI 확보
* 범정부 AI 공통기반 구축

**결론:** 지방정부는 이를 실행하는 단위 조직 역할을 수행합니다.

### 1.2 국가 전략 및 지방 RFP 변환 구조
| 국가 전략 | RFP 요구사항 |
| :--- | :--- |
| AI 행정 혁신 | 행정 업무방식 혁신 |
| 대국민 AI 서비스 | 민원 챗봇 |
| LLM 기반 자동화 | Agent 기반 실행 |
| 데이터 기반 AI | RAG 및 Knowledge Graph |

---

## 2. RFP의 본질적 의도

### 2.1 단순 시스템 구축이 아닌 "AI 운영 체계 구축"
**핵심 구성 요소**
* LLMOps
* RAG
* Agent
* 모델 관리, 평가, Fine-tuning

**결론:** AI를 지속적으로 운영하고 교체 가능한 플랫폼 구축이 목표입니다.

### 2.2 모델 Lock-in 방지 구조
**요구사항 특징**
* 모델 2종 이상 비교
* Fine-tuning 및 재배포
* 외부 LLM 제한적 활용

**의미:** 특정 모델에 종속되지 않고 국가 LLM 교체에 대비하는 구조입니다.

### 2.3 온프레미스 요구의 의미
**요구사항**
* 45B 이상 모델
* 내부망 완전 분리
* 외부 전송 금지

**해석 및 결론:** ChatGPT 등 외부 API 의존을 금지하고 데이터 주권을 확보하는 주권형 AI 구조가 필수적입니다.

## 3. 국가 LLM 연계 구조 (핵심)

### 3.1 현재 구조
* 로컬 LLM (온프레미스)
* RAG 및 Agent
* 내부 데이터 기반

### 3.2 미래 구조
* 국가 LLM (World Best LLM)
* 공공 API 또는 배포형 모델

### 3.3 연결 구조
* LLMOps 통한 모델 교체 가능성 확보
* Orchestrator 통한 모델 Routing
* RAG 통한 모델 독립 구조 확보

**핵심:** 현재는 로컬 LLM을 사용하지만, 미래에는 국가 LLM을 연결할 수 있는 구조입니다.

---

## 4. 의성군이 진짜로 원하는 것

### 4.1 데이터정보센터 활용 문제
* **문제:** 이미 구축되어 있으나 활용도가 낮고 직원들의 데이터 활용 역량이 부족합니다.
* **해결 방향:** RAG 기반 AI 지식베이스로 전환합니다.

### 4.2 AI 플랫폼의 역할
데이터정보센터를 단순 DB에서 지능형 의사결정 시스템(AI Brain)으로 전환합니다.

**활용 영역**
* 정책 데이터
* 사업 관리
* 감사 사례
* 행정 문서

### 4.3 인구 구조 문제 (핵심 동기)
* **주요 지표:** 고령화율 46.7%, 사망자 수가 출생자 수의 6배, 생산가능인구 급감
* **문제:** 공무원 인력 부족 및 업무 증가
* **해결 방안:** Agent 기반 업무 자동화 도입

### 4.4 공모사업 전략
* **목표:** 연 120건, 600억 원 국비 확보
* **의도:** AI 플랫폼을 다음 공모사업 경쟁력 확보를 위한 레퍼런스로 활용합니다.

| 단계 | 주요 내용 |
| :--- | :--- |

| **AI 플랫폼 구축** | 데이터 AI 지식화, Agent 자동화, RAG 및 KG 기반 분석, 행정 효율화 |
| **미래 확장** | 국가 LLM 연계, 전 부서 AI 확대, 데이터 기반 정책 수립 |

## 6. 요구 아키텍처 핵심

### 6.1 망 분리 구조
* **내부 (직원용):** 온프레미스 LLM, 내부 RAG, 보안 중심
* **외부 (대민용):** 클라우드 기반, 공개 데이터 RAG, 챗봇 및 Agent

### 6.2 필수 구성 요소
* Hybrid RAG (Vector 및 Graph 결합)
* Knowledge Graph
* Orchestrator 및 Router
* LLMOps
* Agent 시스템

**특징:** LLM 모델이 변경되어도 전체 시스템 구조는 유지됩니다.

---

## 7. 사업 목적 (단계별)

* **단기:** 민원 챗봇 구축 및 업무 자동화
* **중기:** 데이터 구조화 및 AI 운영 체계 구축
* **장기 (핵심):** 국가 LLM 연계
