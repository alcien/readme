# 의성군 AI 플랫폼 사업의 본질 분석
> (AI G3 도약 기반 + 국가 LLM 연계 전략 + 지역 문제 해결 통합 해석)

---

# 1. 이 사업의 본질 (한 줄 정의)

👉 본 사업은  
**“AI G3 도약 정책을 지방정부에서 구현하면서,  
향후 국가 LLM을 수용할 수 있는 공공 AI 플랫폼을 구축하는 사업”**

---

# 2. 출발점: AI G3 도약 정책

## 2.1 정부 정책 핵심

- AI G3 (Global Top 3) 달성
- 공공부문 AI 전면 도입 (AX)
- 국산 LLM / 주권형 AI 확보
- 범정부 AI 공통기반 구축

👉 RFP는 이 정책을 그대로 반영

---

## 2.2 정책 → RFP 변환 구조

| 국가 정책 | RFP 요구 |
|----------|----------|
| AI 행정 혁신 | 행정 업무 혁신 |
| 대민 AI 서비스 | 민원 챗봇 |
| LLM 자동화 | Agent 기반 실행 |
| 데이터 기반 AI | RAG + Knowledge Graph |

👉 결론  
**정책이 기술 요구사항으로 내려온 상태**

---

# 3. 핵심 의도 ①: “AI 시스템”이 아니라 “AI 운영 체계”

## 3.1 단순 챗봇이 아님

구성 요소:
- LLMOps
- RAG
- Agent
- 모델 관리 / 평가 / Fine-tuning

👉 의미:
> “AI를 계속 운영하고 교체할 수 있는 플랫폼”

---

## 3.2 모델 교체 전제 구조

- 모델 2종 이상 비교
- Fine-tuning 및 재배포
- 외부 LLM 제한적 허용

👉 핵심:
> “LLM은 바뀌고, 시스템은 유지된다”

---

# 4. 핵심 의도 ②: 국가 LLM 연결을 위한 구조 설계

## 4.1 현재 요구

- 온프레미스 LLM (45B 이상)
- 내부망 완전 분리
- RAG + Agent

## 4.2 미래 구조

- 국가 LLM (World Best LLM)
- 공공 API 또는 배포형 모델

---

## 4.3 연결 설계 (핵심)

- LLMOps → 모델 교체 가능
- Orchestrator → 모델 routing
- RAG → 모델 독립 구조

👉 결론:

> **“지금은 로컬 LLM을 쓰지만,  
> 미래에는 국가 LLM으로 교체 가능한 구조”**

---

# 5. 핵심 의도 ③: 주권형 AI (Sovereign AI)

## 5.1 온프레미스 요구의 의미

- 내부 데이터 외부 반출 금지
- 폐쇄망 운영
- 외부 LLM 제한적 사용

👉 의미:

- ChatGPT 의존 금지
- 해외 API 의존 금지
- 데이터 주권 확보

---

## 5.2 구조

### 내부 (직원용)
- 온프레미스 LLM
- 내부 RAG
- 보안 중심

### 외부 (대민용)
- 클라우드 기반
- 공개 데이터 RAG
- 챗봇 + Agent

---

👉 핵심:
> “보안은 내부, 서비스는 외부 → 하이브리드 구조”

---

# 6. 의성군 현실: 왜 이 사업이 필요한가

## 6.1 인구 구조 문제 (가장 중요)

- 고령화율: 46.7%
- 사망 > 출생 (6배)
- 생산가능인구 급감

👉 결과:
- 공무원 인력 부족
- 행정 지속 불가능 구조

---

## 6.2 해결 방향

👉 AI가 인력을 대체

- 인수인계 Agent
- 업무 자동화 Agent
- 민원 대응 자동화

👉 핵심:
> “AI는 선택이 아니라 생존 전략”

---

## 6.3 데이터 문제

현재 상태:
- 데이터정보센터 존재
- 활용도 매우 낮음
- 직원 활용 역량 부족

---

## 6.4 해결 구조

👉 데이터정보센터 → AI Brain

- RAG 기반 지식화
- 자연어 질의 응답
- 정책 데이터 활용

👉 핵심:
> “데이터를 저장에서 활용으로 전환”

---
