# 1. 사업 배경: AI G3 도약과의 직접 연결

## 1.1 정부 정책 기반

의성군 RFP의 추진 배경은 단순 사업이 아니라 **국가 AI 전략의 하위 실행**입니다.

> “정부 ‘AI G3 도약’을 목표로 공공부문 전반의 인공지능 도입”

### 핵심 정책 방향
- AI G3 (Global Top 3) 달성
- 공공부문 AI 전면 도입 (AX: Admin eXperience)
- 국산 LLM 및 주권형 AI 확보
- 범정부 AI 공통기반 구축

→ 지방정부는 이를 **실행하는 단위 조직**

---

## 1.2 국가 전략 → 지방 RFP 변환 구조

| 국가 전략 | RFP 요구사항 |
|----------|-------------|
| AI 행정 혁신 | 행정 업무방식 혁신 |
| 대국민 AI 서비스 | 민원 챗봇 |
| LLM 기반 자동화 | Agent 기반 실행 |
| 데이터 기반 AI | RAG + Knowledge Graph |

👉 즉, **정책 → 기술 요구사항으로 번역된 상태**

# 2. RFP의 본질적 의도

## 2.1 단순 시스템 구축이 아닌 “AI 운영 체계 구축”

핵심 구성:
- LLMOps
- RAG
- Agent
- 모델 관리 / 평가 / Fine-tuning

👉 결론:
> “AI를 지속적으로 운영·교체 가능한 플랫폼 구축”

---

## 2.2 모델 Lock-in 방지 구조

요구사항 특징:
- 모델 2종 이상 비교
- Fine-tuning 및 재배포
- 외부 LLM 제한적 활용

👉 의미:
- 특정 모델 종속 X
- **국가 LLM 교체 대비 구조**

---

## 2.3 온프레미스 요구의 의미

요구:
- 45B 이상 모델
- 내부망 완전 분리
- 외부 전송 금지

👉 해석:
- ChatGPT 등 외부 API 의존 금지
- 데이터 주권 확보

👉 결론:
> “주권형 AI 구조 필수”

# 3. 국가 LLM 연계 구조 (핵심)

## 3.1 현재 구조

- 로컬 LLM (온프레미스)
- RAG + Agent
- 내부 데이터 기반

## 3.2 미래 구조

- 국가 LLM (World Best LLM)
- 공공 API 또는 배포형 모델

## 3.3 연결 구조

- LLMOps → 모델 교체 가능
- Orchestrator → 모델 Routing
- RAG → 모델 독립 구조

👉 핵심:
> “지금은 로컬 LLM, 미래에는 국가 LLM 연결”

# 4. 의성군이 진짜로 원하는 것

## 4.1 데이터정보센터 활용 문제

문제:
- 이미 구축되어 있으나 활용도 낮음
- 직원 데이터 활용 역량 부족

👉 해결 방향:
- RAG 기반 AI 지식베이스 전환

---

## 4.2 AI 플랫폼의 역할

데이터정보센터 → AI Brain

활용 영역:
- 정책 데이터
- 사업 관리
- 감사 사례
- 행정 문서

👉 단순 DB → **지능형 의사결정 시스템**

---

## 4.3 인구 구조 문제 (핵심 동기)

### 주요 지표
- 고령화율: 46.7%
- 사망 > 출생 (6배)
- 생산가능인구 급감

👉 문제:
- 공무원 인력 부족
- 업무 증가

👉 해결:
- Agent 기반 업무 자동화

---

## 4.4 공모사업 전략

목표:
- 연 120건 / 600억 확보

의도:
- AI 플랫폼 = **레퍼런스 확보 수단**

👉 핵심:
> “AI 사업 → 다음 공모사업 경쟁력 확보”
