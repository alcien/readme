# 내부직원 업무지원 AI 비서 개발 요구사항 정의서 (RD)

**문서 버전:** v0.1  
**기준 문서:** 국내 공공기관 내부직원 업무지원 AI 비서 운영 현황 분석 보고서  
**적용 대상:** 공공기관 내부 공무원·직원 대상 업무지원 AI Agent (챗봇 2.0)

---

## 1. 개요 및 목적

### 1.1 배경 및 범위
본 요구사항 정의서는 공공기관 내부 직원이 업무망(내부망) 환경에서 직접 사용하는 AI 업무지원 비서(AI Agent) 시스템 개발에 필요한 기능·보안·UI/UX 요구사항을 정의한다. 단순 키워드 검색 챗봇(1.0)의 한계를 극복하고, 자연어 기반 멀티턴 대화로 복수의 내부 행정 시스템을 통합 조작하는 대화형 게이트웨이(챗봇 2.0) 구현을 목표로 한다.

### 1.2 참조 시스템 및 사례

| 참조 기관·시스템 | 참조 목적 |
| :--- | :--- |
| **경기도 AI 행정비서** | 독립망 구축, 온프레미스 LLM 운영 구조 |
| **K-water 업무용 생성형 AI** | sLLM + RAG 아키텍처 설계 표준 |
| **서울시 생성형 AI 활용 실태** | 사용자 규모, 도입 우선순위, 섀도우 AI 대응 |
| **서울 중구 AI내편중구** | 통합 데이터베이스 기반 대화형 게이트웨이 UX |
| **LH 스마트 계측시스템** | 도메인 특화 현장 AI, 실시간 데이터 연동 |
| **DARVIS (비즈니스PT)** | XAI 기반 근거 문서 추적 UI, 내부 규정 검색 |
| **조달청 생성형AI업무지원서비스** | 보안 요건 기준선 (국정원 '상' 등급) |

## 2. 시스템 기능 요구사항 (Functional Requirements)

### 2.1 기능 영역 분류
* **FR-01** 자연어 기반 지식 검색 및 규정 조회
* **FR-02** 문서 처리·요약·초안 자동 생성
* **FR-03** 데이터 분석 및 통계 시각화
* **FR-04** 내부 행정 시스템 통합 게이트웨이 (Agent 핵심)
* **FR-05** 도메인 특화 현장 데이터 분석 지원
* **FR-06** 멀티턴 대화 및 작업 컨텍스트 유지
* **FR-07** XAI 기반 답변 근거 추적 및 검증
* **FR-08** RAG 기반 내부 지식베이스 관리
* **FR-09** 사용자 및 권한 관리
* **FR-10** 보안·망분리 준수 아키텍처

## 3. UI/UX 요구사항 정의서

> **벤치마크:** 서울 중구 AI내편중구 (통합 게이트웨이형 UI), 경기도 AI 행정비서 (온프레미스 챗봇), GitHub Copilot Chat / Notion AI (업무지원 AI Agent UX 표준)

### 3.1 전체 UI 철학
| 원칙 | 정의 |
| :--- | :--- |
| **게이트웨이 우선 (Gateway-first)** | 모든 내부 시스템 접근의 시작점이 AI 대화 창. 메뉴 트리 탐색이 아닌 명령 입력이 주 인터랙션 |
| **투명한 AI (Transparent AI)** | 모든 답변에 근거 출처가 명시되어 공무원이 AI 답변을 그대로 수용하지 않고 검증할 수 있는 구조 |
| **작업 중심 레이아웃 (Task-oriented)** | 대화 창 중심이되, 생성된 문서·차트·결과물이 사이드패널 또는 인라인으로 즉시 확인 가능 |
| **중단 가능성 보장 (Interruptible)** | 잘못된 방향으로 진행되는 AI 생성을 언제든 즉시 중단 가능 |
| **감사 추적 가시화 (Auditable)** | 사용자가 수행한 작업·실행된 시스템 액션의 이력이 항상 조회 가능한 형태로 존재 |
